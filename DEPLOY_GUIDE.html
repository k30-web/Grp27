Deploy to GitHub Pages

Go from files on your computer to a live public URL in under 5 minutes

🌐 Free hosting · Custom URL · Auto-deploys on every push

Part 1 — One-Time Setup

1
📁 Create a GitHub repository

Sign in to github.com. Click the + icon (top right) → New repository.

Fill in the form:

github.com / new
Repository name:ayurveda-wellness
Visibility:⦿ Public
Initialize:☐ Leave unchecked (we'll push files manually)

Click Create repository. Copy the repo URL — you'll need it in Step 3.

2
⚙️ Enable GitHub Pages

In your new (empty) repo, go to Settings → Pages.

Under Build and deployment → Source, choose:

Settings → Pages
Source:⦿ GitHub Actions

Click Save. This tells GitHub to use our workflow file to deploy automatically.

Why GitHub Actions? It lets us control exactly what gets deployed and auto-deploys every time you push a change — no manual steps needed after setup.
3
💻 Install Git (if you don't have it)

Check if Git is installed:

git --version

If not installed, download from git-scm.com and install with default settings.

Then configure your identity (one-time setup):

# Replace with your real name and GitHub email
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
4
📂 Prepare your project folder

Download the project ZIP from Claude (or locate the files you already have). Your folder should look like this:

ayurveda-wellness/
├── index.html ← The entire app
├── .github/
│ └── workflows/
│ └── deploy.yml ← Auto-deploy workflow
├── README.md ← Shown on GitHub
├── LICENSE ← MIT license
└── .gitignore ← Files to ignore
Important: The .github/workflows/deploy.yml file is what makes auto-deploy work. Don't rename or move it.
5
🚀 Push files to GitHub

Open Terminal (Mac/Linux) or Command Prompt / PowerShell (Windows). Navigate to your project folder, then run:

# 1. Go into the project folder
cd ayurveda-wellness

# 2. Initialize git
git init

# 3. Connect to your GitHub repo (paste YOUR repo URL)
git remote add origin https://github.com/YOUR-USERNAME/ayurveda-wellness.git

# 4. Stage all files
git add .

# 5. Create first commit
git commit -m "🌿 Initial deployment of Ayurveda Wellness Chatbot"

# 6. Push to GitHub (triggers auto-deploy!)
git branch -M main
git push -u origin main
✅ After this push, GitHub Actions automatically builds and deploys your app. You don't need to do anything else.
6
⏳ Wait ~60 seconds, then visit your live URL

Go to your repo on GitHub → click the Actions tab. You'll see a workflow run in progress (yellow spinner). Once it turns green ✅, your app is live.

Your permanent URL is:

🌐
Your live app URL
https://YOUR-USERNAME.github.io/ayurveda-wellness/

Replace YOUR-USERNAME with your actual GitHub username.

First deployment takes ~60 seconds. After that, every push deploys in under 30 seconds.

Part 2 — Making Updates Later

✎
✏️ Edit → Push → Done

Whenever you want to update the app (change content, colors, add questions), just edit index.html in any text editor, then:

# Run these 3 commands from your project folder
git add .
git commit -m "Update: describe what you changed"
git push

GitHub Actions picks it up automatically and your live site updates within ~30 seconds. No manual deploy step needed — ever.

Part 3 — Optional Extras

🔗 Custom Domain (e.g. ayurveda.yourdomain.com)
  • Go to Settings → Pages → Custom domain → enter your domain
  • Add a CNAME DNS record at your domain registrar pointing to YOUR-USERNAME.github.io
  • Check "Enforce HTTPS" after the DNS propagates (~10 min)
🔒 Update README badges

Open README.md and replace every YOUR-USERNAME with your real GitHub username. This makes the "Live Demo" badge and deploy badge link to the right URLs.

🌟 Star your own repo

GitHub shows stars on repo pages — and it's a nice way to bookmark it. Hit the ⭐ Star button on your repo page!

Troubleshooting

❌ "Permission denied" on git push

GitHub no longer accepts passwords for pushes. Use a Personal Access Token: go to GitHub → Settings → Developer Settings → Personal access tokens → Generate new token. Check repo scope. Use this token as your password when prompted.

❌ Action fails with "pages not enabled"

Go back to Settings → Pages → Source and confirm it's set to GitHub Actions (not "Deploy from branch"). Save and re-run the workflow from the Actions tab.

❌ Site shows 404 after deploy

Ensure the file is named exactly index.html (lowercase). GitHub Pages serves index.html automatically for the root URL. Wait another 60 seconds and hard-refresh (Ctrl+Shift+R).

❌ Yellow spinner never goes green

Click the workflow run in the Actions tab to see the error log. The most common issues are: wrong branch name (must be main) or deploy.yml is in the wrong folder (must be .github/workflows/deploy.yml).
